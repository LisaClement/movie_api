Documentation


Objective

This web application provides users with access to information about different movies, directors, and genres. Users are able to sign up, update their personal information, and create a list of their favorite movies.


Features

  • Return a list of ALL movies to the user
  • Return data (description, genre, director, image URL)
  • Return data about a genre (description) by name/title (e.g "Thriller")
  • Return data about a director by name
  • Allow new users to register
  • Allow users to update their user info
  • Allow users to add a movie to their list of favorites
  • Allow users to remove a movie from their list of favorites
  • Allow existing users to deregister
  • Allow users to create a "To Watch" list in addition to their "Favorite Movies" list

Business Logic URL HTTP Methode Request body data format Response body data format
Return a list of ALL movies to the user /movies GET None A JSON object holding data about all the movies
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user /movies/[title] GET None A JSON object holding data about a specified movie
Return a list of movies from a specific genre movies/genres/[genre] GET None A JSON object holding a list of movies from a specific genre.
Return a list of movies from a specific director /movies/directors/[name] GET None A JSON object holding a list of movies from a specific director
Allow new users to register /users/register POST A JSON object holding data for a new user.
Example:{
"Username": "Elisabeth"
"Email": "eli@gmail.com"} 		A JSON object holding data for the new user (email, username)
Allow users to update their user info (username) /users/[userID] PUT A JSON object holding the data to be updated for the user. A text message indicating the email/username combination that was updated
Allow users to add a movie to their list of favorites (showing only a text that a movie has been added—more on this later) /users/[username]/favorites/add/[title] POST None A message indicating that a movie was successfully added to a user’s favorite list.
Allow existing users to deregister (showing only a text that a user email has been removed—more on this later) /users/deregister/[username] DELETE None A message indicating that a user was removed.
